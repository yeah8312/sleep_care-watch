# Sleep Care 프로젝트 계획

## 문서 목적
이 문서는 프로젝트 전체 방향을 빠르게 공유하기 위한 통합 계획 문서다. 세부 구현 기준은 하위 문서와 [통신 및 판단 아키텍처](./sleepcare_protocol_design.md)를 우선 참조한다.

## 현재 구현 상태
- 모바일 앱은 Pi와의 로컬 네트워크 `NSD + WSS` 연동을 구현했다.
- 모바일 앱은 공부 세션 시작/종료, 세션 타이머, Pi 실시간 위험 상태와 알림 저장을 지원한다.
- 워치 앱은 아직 구현되지 않았으며, 앱에서는 "워치 앱 준비 중" 상태를 사용한다.
- 실제 수면 데이터 수집은 아직 연결되지 않았고, 수면 분석/추천은 empty state와 보정 없음 기준으로 동작한다.

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
- 수험생의 학습 중 졸음 상태를 감지한다.
- 사용자의 실제 수면 패턴을 기록하고 분석한다.
- 졸음 데이터와 수면 데이터를 함께 분석해 더 나은 수면 스케줄을 제안한다.
- 사용자가 모바일 앱에서 자신의 상태를 쉽게 확인하고 행동으로 옮길 수 있게 한다.

### 1.2 핵심 사용자
- 장시간 책상에서 공부하는 수험생
- 수면 부족 또는 불규칙한 생활 패턴으로 학습 효율이 떨어지는 사용자
- 시험 일정에 맞춰 수면 습관을 조정하고 싶은 사용자

### 1.3 핵심 가치
- 실시간 졸음 감지
- 수면 패턴 기반 분석
- 시험 일정 및 학습 계획 기반 수면 스케줄 제안
- 디바이스 간 연동을 통한 일상 밀착형 케어

## 2. 전체 제품 구성

### 2.1 라즈베리파이 디바이스
- 카메라를 통해 사용자의 안구 상태를 감지한다.
- 학습 세션 중 졸음 위험도를 계산한다.
- 졸음 감지 시 LCD와 스피커로 즉시 피드백을 제공한다.
- 모바일 앱으로 세션 이벤트를 전송한다.

### 2.2 모바일 앱
- 전체 세션의 제어 허브 역할을 담당한다.
- 라즈베리파이 이벤트를 수집하고 저장한다.
- 수면 기록과 학습 세션 기록을 통합 분석한다.
- 수면 스케줄과 생활 가이드를 제안한다.

### 2.3 워치 앱
- 학습 세션 중 심박/IBI 데이터를 수집한다.
- 모바일 앱과 실시간 동기화한다.
- 필요 시 진동 보조 경고를 수행한다.
- 수면 데이터 연동 경로를 지원한다.
- 현재는 미구현 상태다.

## 3. 기본 사용자 흐름

1. 사용자는 모바일 앱에서 프로필, 시험 일정, 학습 플랜을 설정한다.
2. 모바일 앱은 라즈베리파이를 같은 로컬 Wi-Fi에서 찾는다.
3. 사용자가 공부 세션을 시작하면 라즈베리파이는 안구 상태를 감지한다.
4. 모바일 앱은 Pi 이벤트를 저장하고 세션 상태를 갱신한다.
5. 라즈베리파이는 안구 데이터 기준으로 졸음 위험도를 계산하고 즉시 경고한다.
6. 모바일 앱은 수면 기록이 없더라도 졸음 패턴, 일정, 목표를 바탕으로 수면 스케줄과 생활 가이드를 제안한다.

## 4. 시스템 구조 기준

### 4.1 통신 구조
- 워치 ↔ 모바일 앱: Wear OS Data Layer
- 모바일 앱 ↔ 라즈베리파이: 로컬 네트워크(NSD + WSS)
- 수면 데이터 연동: 모바일 앱 기준 통합 경로를 두고 워치 앱은 연동 상태 표시와 유도 역할을 맡는다.

### 4.2 기기별 역할 분리
- 라즈베리파이: 실시간 감지와 즉시 반응
- 워치: 심박/IBI 수집, 진동 보조 경고, 수면 데이터 연동 보조
- 모바일 앱: 세션 제어, Pi 데이터 수신, 저장, 분석, 추천, 사용자 인터페이스

### 4.3 모바일 앱 기술 스택 방향
- 앱 본체: Kotlin
- UI: Jetpack Compose + Material 3
- 아키텍처: MVVM + Repository + UDF(단방향 데이터 흐름)
- 상태관리 / 비동기 / DI: ViewModel + Kotlin Coroutines + Flow + Hilt
- 로컬 저장: Room + DataStore
- 백그라운드 작업: WorkManager

## 5. 프로그램별 계획 요약

### 5.1 라즈베리파이
- 카메라 영상 수집
- 얼굴 및 눈 상태 감지
- 규칙 기반 졸음 감지 MVP
- 심박 보조 신호 수신
- 로컬 경고 출력
- NSD + WSS 기반 연동

### 5.2 모바일 앱
- 기기 연결과 세션 시작/종료
- Pi 위험도/알림 수신 및 저장
- 세션 로그 저장과 분석 화면 제공
- 수면 기록 empty state 표시
- 수면 스케줄 제안과 리마인더 제공

### 5.3 워치 앱
- 갤럭시 워치 대상 심박/IBI 수집
- Samsung Health Sensor SDK 기반 센서 파이프라인
- Wear OS Data Layer 기반 전달
- 진동 보조 경고
- 수면 데이터 연동 상태 표시 및 모바일 앱 유도
- 현재는 미구현 상태

## 6. 데이터 및 추천 계획

### 6.1 핵심 데이터
- 학습 세션 데이터
- 졸음 이벤트 데이터
- 심박 데이터
- 수면 데이터
- 사용자 입력 데이터

### 6.2 추천 로직 방향
- 규칙 기반 추천으로 시작한다.
- 최근 수면 시간과 규칙성을 분석한다.
- 졸음 발생 시간대와 수면 패턴의 상관관계를 본다.
- 시험 일정과 학습 플랜을 반영해 권장 취침/기상 시간을 제안한다.
- 실시간 심박 데이터는 세션 해석 보조 지표로 활용한다.
- 수면 데이터가 없을 때도 추천이 동작해야 한다.

## 7. 단계별 우선순위

### 7.1 1단계: MVP
- 라즈베리파이 기본 졸음 감지와 현장 경고
- 모바일 앱 데이터 수집 및 조회
- 워치 없는 `eye-only` 세션 지원
- 수면 데이터가 없어도 동작하는 간단한 수면 스케줄 제안

### 7.2 2단계: 안정화
- 세션 재전송과 복구 로직 강화
- 리포트와 시각화 고도화
- 시험 일정 및 학습 플랜 기반 추천 강화

### 7.3 3단계: 확장
- 사용자 개인화
- 장기 패턴 분석
- 추가 생체 신호 활용 검토
- 클라우드 동기화 검토

## 8. 주요 의사결정 포인트와 리스크

### 8.1 의사결정 포인트
- 라즈베리파이 안구 감지 MVP 기준
- 세션 데이터 저장 범위
- 수면 데이터 모바일 통합 저장 스키마

### 8.2 주요 리스크
- 졸음 감지 정확도 부족
- 워치 플랫폼별 API 차이
- 네트워크 단절과 재전송 복잡도
- 수면 데이터 접근 제한

### 8.3 대응 방향
- 감지 정확도는 규칙 기반에서 단계적으로 개선한다.
- 플랫폼 제약은 초기에 검증한다.
- 모든 이벤트에 시각 정보와 세션 ID를 저장한다.
- 실시간 세션과 장기 추천 데이터를 분리해 설계한다.
