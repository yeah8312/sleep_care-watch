# 워치 앱 구현 계획

## 문서 목적
- 갤럭시 워치 기반 워치 앱 MVP를 바로 구현할 수 있도록 범위와 결정을 고정한다.
- 실시간 학습 세션용 심박/IBI 파이프라인과 수면 데이터 연동 경로를 워치 앱 기준으로 정리한다.
- 세부 메시지 구조는 [통신 및 판단 아키텍처](./sleepcare_protocol_design.md)를 우선 참조하고, 이 문서는 워치 앱 구현 책임과 화면/상태/제약을 정의한다.

## 확정 사항
- 1차 대상 기기는 갤럭시 워치다.
- 학습 세션 중 심박/IBI 수집 SDK는 Samsung Health Sensor SDK를 사용한다.
- 워치 ↔ 모바일 실시간 통신은 Wear OS Data Layer를 사용한다.
- 수면 데이터 연동은 프로젝트 범위에 포함한다.
- 수면 데이터의 1차 수집/통합 책임은 모바일 앱에 두고, 워치 앱은 세션 중 실시간 센서 수집과 수면 연동 상태 표시/유도에 집중한다.

## 현재 구현 상태
- 워치 앱은 아직 구현되지 않았다.
- 모바일 앱은 워치 연결을 시도하지 않고 "워치 앱 준비 중" 상태를 표시한다.
- 프로토콜 문서에 정의된 `hr.ingest`, 진동 보조 경고, 수면 데이터 전달은 현재 앱에서 사용하지 않는다.

## 목표
- 학습 세션 중 안정적으로 심박/IBI를 수집해 모바일 앱으로 전달한다.
- 모바일 앱에서 시작한 세션을 워치가 따라가며 연결 상태를 분명하게 보여준다.
- Pi 경고 시 워치 진동 보조 경고를 수행한다.
- 사용자가 수면 데이터 연동 상태를 이해하고 필요한 권한/설정을 완료할 수 있게 한다.

## 워치 앱 MVP 범위

### 포함
- 세션 시작/종료에 따라 심박/IBI 수집 시작 및 중지
- Data Layer 기반 live/batch 심박 전송
- 모바일 ACK 기준 재전송 및 간단한 백필
- Pi 경고를 모바일에서 전달받아 진동 수행
- 워치 연결 상태, 센서 상태, 마지막 동기화 상태 표시
- 수면 데이터 연동 상태 화면과 모바일 앱 열기/설정 유도

### 제외
- 워치 단독 졸음 판단
- 워치에서 장기 분석/리포트 계산
- 워치에서 수면 원시 데이터 장기 저장
- 심박 외 추가 생체 신호 연동
- LTE 단독 모드 최적화

## 역할 분리

### 워치 앱
- Samsung Health Sensor SDK로 심박/IBI를 읽는다.
- 세션 중 샘플을 버퍼링하고 모바일 앱으로 전달한다.
- 모바일에서 전달한 flush 정책과 경고 명령을 적용한다.
- 센서/권한/연결 상태를 UI에 표시한다.

### 모바일 앱
- 세션 시작/종료의 소스 오브 트루스 역할을 한다.
- 워치로 제어 메시지를 보내고 수신 심박을 Pi에 `hr.ingest`로 중계한다.
- 수면 데이터 연동의 실제 저장/조회 주체가 된다.

### 라즈베리파이
- 안구 데이터와 심박 데이터를 융합해 최종 위험도를 계산한다.
- `risk.update`, `alert.fire`, `session.summary`를 모바일에 전송한다.

## 구현 기준

### 앱 구조
- `wear` 모듈 단일 앱으로 시작한다.
- 워치 UI는 Compose for Wear OS를 사용한다.
- 아키텍처는 모바일과 동일하게 `ViewModel + Repository + UDF`를 따른다.
- 센서 수집, Data Layer 전송, 진동 제어는 각각 별도 data source로 분리한다.

### 최소 화면
- 연결 대기 화면: 폰 연결 상태, 세션 대기 상태 표시
- 세션 실행 화면: 현재 세션 상태, 센서 수집 상태, 마지막 전송 시각 표시
- 경고 상태 화면: 최근 경고 레벨과 진동 수행 결과 표시
- 설정/상태 화면: 권한 상태, 센서 지원 여부, 수면 연동 안내, 모바일 앱 열기 액션 제공

### 세션 상태 머신
- `IDLE`: 세션 없음, 센서 비활성
- `WAITING_PHONE`: 워치 앱은 켜져 있으나 시작 명령 대기
- `STARTING`: 권한/센서 준비 및 버퍼 초기화
- `RUNNING`: 심박/IBI 수집 및 live/batch 전송
- `DEGRADED`: 센서 품질 저하 또는 폰 연결 끊김으로 batch 위주 동작
- `ALERTING`: 진동 경고 수행 중
- `STOPPING`: 세션 종료 처리와 최종 flush
- 오류 시 `ERROR_PERMISSION`, `ERROR_SENSOR`, `ERROR_LINK` 상태를 별도 표기한다.

### 센서 수집 규칙
- 세션이 `RUNNING` 상태일 때만 센서를 활성화한다.
- 기본 수집 목표는 BPM과 가능한 경우 IBI다.
- 샘플에는 `sample_seq`, `sensor_ts_ms`, `bpm`, `hr_status`, `ibi_ms`, `ibi_status`를 포함한다.
- 품질이 낮은 샘플도 버리지 않고 상태값과 함께 전송한다.
- 센서 초기화 직후의 불안정 구간은 `busy_or_initial`로 라벨링해 전달한다.

### flush 정책
- 기본값은 `NORMAL=15초`, `SUSPECT=5초`, `ALERT_NEAR=2초`, `RECOVERY=15초`를 따른다.
- 워치는 모바일에서 받은 최신 flush 정책을 메모리 상태로 유지한다.
- live 전송은 최신 샘플 1건 기준, batch 전송은 ACK되지 않은 샘플 묶음 기준으로 보낸다.
- 폰 연결이 잠시 끊기면 로컬 메모리 버퍼에 적재하고 재연결 시 순서대로 재전송한다.
- MVP에서는 디스크 영속 버퍼를 두지 않고 메모리 버퍼만 사용한다.

### ACK 및 재전송
- 모바일은 `hr.ack.body.ack_sample_seq`로 누적 ACK를 반환한다.
- 워치는 ACK 이전 샘플을 버퍼에서 제거하지 않는다.
- 같은 `sample_seq`를 재전송할 수 있으며 모바일/Pi는 중복을 허용하는 방향으로 구현한다.
- 워치 재시작 후 미전송 샘플 복구는 MVP 범위에서 제외한다.

### 진동 경고
- 모바일에서 `/sc/v1/alert/vibrate`를 받으면 즉시 진동을 수행한다.
- 패턴 문자열은 쉼표 구분 millisecond 목록으로 해석한다.
- 워치가 `Do Not Disturb` 또는 시스템 제약으로 진동 수행에 실패하면 UI에 실패 상태를 남긴다.
- 경고가 끝나면 별도 유지 화면 없이 최근 경고 이력만 보여준다.

### 수면 데이터 연동
- 1차 구현에서 워치 앱은 수면 세션 원시 데이터를 직접 저장하지 않는다.
- 수면 데이터의 실제 조회/저장은 모바일 앱에서 처리한다.
- 워치 앱은 사용자가 수면 데이터 연동을 완료할 수 있도록 다음만 제공한다.
- 현재 연동 상태 표시
- 연동 필요 시 모바일 앱 열기
- 최근 동기화 시각 표시
- 연동 미완료 시 "학습 세션 심박은 동작하지만 수면 분석은 제한됨" 안내 표시

### 권한 및 사용자 안내
- 첫 실행 시 센서 권한과 백그라운드 측정 제약을 단계적으로 안내한다.
- 권한이 없으면 세션 시작을 허용하지 않고 권한 요청 화면으로 보낸다.
- 센서 미지원 기기에서는 설치는 가능하되 세션 시작을 막고 "지원되지 않는 기기" 문구를 표시한다.

### 배터리 및 안정성 기준
- 평상시에는 세션 중에만 센서를 활성화한다.
- 화면이 꺼져도 세션 동안 수집이 유지되어야 한다.
- 연결이 불안정하면 live보다 batch 전송을 우선한다.
- 센서 미수집이 30초 이상 지속되면 사용자에게 착용/센서 상태를 확인하라는 메시지를 보여준다.

## 워치 앱과 모바일 앱 계약

### 모바일 → 워치
- 세션 시작: `sid`, `study_mode`, `flush_policy`, `hr_required`, `watch_vibration_enabled`
- 세션 종료: `sid`
- flush 정책 변경: `normal_sec`, `suspect_sec`, `alert_sec`
- 진동 경고: `level`, `pattern`

### 워치 → 모바일
- live 심박: 최신 샘플 1건
- batch 심박: ACK 안 된 샘플 배열
- 상태 동기화: 현재 세션 상태, 마지막 샘플 시각, 마지막 ACK 시각
- 오류 보고: 권한 없음, 센서 없음, 링크 끊김, 진동 실패

## 구현 순서
1. Wear 앱 모듈 생성과 Compose for Wear OS 기본 구조 구성
2. Data Layer 제어 메시지 수신기와 상태 저장소 구성
3. Samsung Health Sensor SDK 기반 심박/IBI 수집기 구현
4. live/batch 전송, ACK 처리, 메모리 버퍼 구현
5. 세션 실행 화면과 상태 화면 구현
6. 진동 경고 처리 및 최근 경고 이력 UI 구현
7. 수면 연동 상태 화면과 모바일 앱 열기 액션 구현
8. 모바일 앱의 `hr.ingest` 연결과 통합 테스트

## 테스트 기준

### 단위 테스트
- flush 정책 변경 시 다음 전송 주기가 올바르게 바뀌는지 확인
- ACK 수신 후 버퍼 정리가 올바른지 확인
- 품질 라벨 매핑이 프로토콜 규칙과 일치하는지 확인
- 진동 패턴 문자열 파싱이 올바른지 확인

### 통합 테스트
- 모바일에서 세션 시작 시 워치가 수집 상태로 전환되는지 확인
- `RUNNING` 중 live/batch 샘플이 모바일로 전달되는지 확인
- ACK 누락 상황에서 재전송이 발생하는지 확인
- 모바일 경고 수신 시 워치 진동이 동작하는지 확인
- 폰 연결 복구 후 누락 샘플 batch 재전송이 되는지 확인

### 수동 테스트
- 워치를 실제 착용한 상태에서 20분 이상 세션 유지
- 화면 꺼짐/켜짐 반복 중 수집 지속 여부 확인
- 블루투스 끊김 후 재연결 시 세션 복귀 여부 확인
- 센서 권한 거부/허용 흐름 확인
- 수면 연동 미완료/완료 상태 문구 확인

## 완료 기준
- 모바일이 시작한 세션을 워치가 따라가며 심박/IBI를 1차 수집한다.
- 모바일은 워치 심박을 받아 Pi로 `hr.ingest`를 보낸다.
- Pi 경고 시 워치 진동이 보조 경고로 동작한다.
- 사용자는 워치에서 연결 상태, 센서 상태, 수면 연동 상태를 확인할 수 있다.
- 수면 연동이 미완료여도 학습 세션 기능은 동작한다.

## 후속 확장
- 디스크 기반 백필 버퍼 추가
- 수면 단계/HRV 등 추가 지표 활용
- 워치 타일 또는 complication 추가
- 워치 단독 간단 경고 정책 검토
