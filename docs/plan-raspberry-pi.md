# 라즈베리파이 계획

## 목표
- 사용자의 졸음 상태를 실시간으로 감지한다.
- 졸음이 감지되면 즉시 알림을 제공한다.
- 모바일 앱과 안정적으로 연동해 학습 세션 판단을 지원한다.

## 주요 기능
- 카메라 영상 수집
- 얼굴 또는 눈 상태 감지
- 졸음 판단 로직
- 심박 데이터 수신 및 보조 반영
- LCD 상태 표시
- 스피커 경고음 재생
- 로컬 네트워크 기반 데이터 송수신
- 기기 상태 확인용 간단 UI

## 모바일 연동 기준
- 모바일 앱과의 통신은 로컬 네트워크(NSD + WSS) 기준으로 맞춘다.
- 전송 데이터는 졸음 이벤트와 위험도 업데이트 중심으로 설계한다.
- 연결이 끊겨도 기기 자체 경고는 지속할 수 있어야 한다.
- 현재 모바일 앱 구현 범위는 `hello`, `session.open`, `risk.update`, `alert.fire`, `session.close`, `session.summary`, `ping/pong` 이다.
- 워치 앱이 아직 없으므로 초기에 `eye-only` 모드만으로도 세션이 정상 동작해야 한다.

## 세부 모듈
- 영상 입력 모듈
- AI 추론 모듈
- 졸음 이벤트 판단 모듈
- 심박 데이터 수신 모듈
- 알림 출력 모듈
- 네트워크 통신 모듈
- 기기 설정 및 로그 모듈

## 구현 단계
1. 카메라 입력 안정화
2. 얼굴 및 눈 상태 감지 구현
3. 규칙 기반 졸음 감지 MVP 구현
4. 모바일 앱의 `eye-only` 세션과 호환되는 네트워크 프로토콜 구현
5. LCD 출력 및 경고음 연결
6. NSD + WSS 기반 데이터 전송 구현
7. 심박 입력 수신 구조 추가
8. 실제 환경 테스트 후 모델 정교화

## 고려 사항
- 조명 변화 대응
- 안경 착용 사용자 대응
- 장시간 실행 안정성
- 발열 및 성능 관리
- 네트워크 단절 시 로컬 동작 보장

## 협업 논의 포인트
- 카메라 위치와 시야각 설계
- 경고 강도와 단계 정의
- 로컬 UI 범위
- 심박 보조 신호의 반영 방식
- 참고 의견: 초기 버전은 경량 모델 또는 규칙 기반 판별부터 시작하는 편이 현실적이다.
- 참고 의견: 스피커는 즉각적인 각성 효과 측면에서 유리하므로 우선순위를 높게 두는 편이 좋다.
